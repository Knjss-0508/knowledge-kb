# 答疑知识中台交付验收清单

更新日期：2026-07-22

## 一、基础验收

- [ ] 代码包不包含 `.env`、密钥、Cookie和真实账号。
- [ ] 代码包不包含真实会话、数据库、日志和 `outputs`。
- [ ] `checksums.sha256` 校验通过。
- [ ] 第三部分测试全部通过。
- [ ] CZ 后端测试全部通过。
- [ ] `scripts/verify_release.ps1`全部步骤通过。
- [ ] Docker Compose 语法检查通过。
- [ ] `/health` 返回正常。
- [ ] `/ready` 能正确反映数据库和 Embedding 状态。

## 二、品类验收

每个品类至少准备2条脱敏记录：

- [ ] 手机
- [ ] 平板
- [ ] 笔记本
- [ ] 相机机身
- [ ] 相机镜头
- [ ] 耳机
- [ ] 手表
- [ ] 游戏机
- [ ] 手写笔
- [ ] 学习机

验证：

- [ ] 中文名称能够映射到稳定编码。
- [ ] 别名能够映射到正确品类。
- [ ] 相机机身和相机镜头不会被错误合并。
- [ ] 不同品类不会自动合并主题。
- [ ] 未知品类进入人工确认。
- [ ] 未知品类不会默认归入手机。

## 三、端到端验收

- [ ] 第二部分可以通过批量接口提交脱敏记录。
- [ ] 单批100条限制生效。
- [ ] 未脱敏记录被拒绝。
- [ ] 同一幂等键重复提交返回复用，不重复调用模型。
- [ ] 第三部分成功拆分原子问题。
- [ ] 同品类内完成1～N主题聚类。
- [ ] 生成10项无标准引用知识草稿、推荐回复和模型初标。
- [ ] 模型初标包含“是否值得沉淀”，值为值得沉淀、不值得沉淀或待确认。
- [ ] 组员可以标注“是否值得沉淀”，不值得沉淀的知识不会进入批量送审。
- [ ] `关联标准项`字段始终保留，新生成的无标准候选默认不填写。
- [ ] 已有关联标准项和来源版本不会被清空或覆盖。
- [ ] 知识ID、主标题、知识内容、推荐回复、知识分类、适用范围和关键词缺失时不能进入 CZ 发布审核。
- [ ] 知识内容或推荐回复包含显式标准引用时原文保留，并进入“标准关联搁置”，不走当前无标准送审入口。
- [ ] 候选进入 CZ 主题审核队列。
- [ ] 审核修改、通过、驳回和Bad Case均能保存。
- [ ] 查重服务可返回通过、复核或阻断结果。
- [ ] 终审通过后知识进入发布状态。
- [ ] 发布知识可以通过搜索接口召回。
- [ ] 召回质量事件可以写回。

## 四、异常验收

- [ ] MiMo未配置时进入规则降级和人工审核。
- [ ] MiMo超时不会丢失原始记录。
- [ ] Embedding不可用时系统明确提示，不伪造查重结果。
- [ ] Qwen3不可用时返回`DEDUP_UNAVAILABLE`且不创建知识。
- [ ] 单条入库后置处理失败不影响下一条成功候选。
- [ ] 图例URL可下载时保存为CZ媒体，媒体ID可保留，明确依赖但不可下载时阻止提交。
- [ ] 批量重复幂等键只创建一条知识，重试返回复用。
- [ ] 证据不足时不生成确定性结论。
- [ ] 任一阶段失败都有状态、错误和可重试入口。

## 五、自动审核验收

- [ ] 初次交付保持 `AUTO_REVIEW_ENABLED=false`。
- [ ] 每个拟自动审核品类均有独立验证样本。
- [ ] 样本量达到业务负责人确认的门槛。
- [ ] 整体准确率达到门槛。
- [ ] 自动放行精确率达到门槛。
- [ ] 错误放行样本已经复盘。
- [ ] 自动审核绑定具体模型版本。
- [ ] 自动审核绑定具体Prompt版本。
- [ ] 自动审核可以按品类单独关闭和回滚。

## 六、交付签收

- [ ] 业务负责人完成操作验收。
- [ ] 第二部分负责人完成接口验收。
- [ ] CZ负责人完成审核发布验收。
- [ ] 运维负责人完成部署、监控和备份验收。
- [ ] 数据负责人确认脱敏与数据权限。
- [ ] 技术负责人确认版本、回滚和已知风险。
