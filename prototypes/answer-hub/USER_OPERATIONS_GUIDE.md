# 答疑知识中台业务用户操作手册

更新日期：2026-07-22

## 1. 适用对象

本手册面向：

- 知识审核员。
- 业务组员。
- 知识运营负责人。
- 质量验收负责人。

业务用户通过浏览器使用系统，不需要安装 Python、Docker或模型服务。

## 2. 用户最终收到什么

业务用户正式收到：

1. 工作台访问地址。
2. CZ 知识库访问地址。
3. 个人账号和角色权限。
4. 本操作手册。
5. 当前生效品类和审核口径。
6. 脱敏数据模板、聚类验证模板和反馈模板。
7. 已知限制和问题反馈渠道。

不要通过聊天、邮件或 Excel 传递 API Key、数据库密码和服务间集成密钥。

## 3. 当前品类

首批配置品类：

- 手机
- 平板
- 笔记本
- 相机机身
- 相机镜头
- 耳机
- 手表
- 游戏机
- 手写笔
- 学习机

相机机身和相机镜头是两个独立品类，不应合并审核。

## 4. 日常操作流程

### 4.1 查看自动化运行

进入“自动化看板”后检查：

- 运行是否成功。
- 入选记录数量。
- 排除记录数量。
- 原子问题数量。
- 主题候选数量。
- 证据不足数量。
- 模型失败和规则降级数量。

系统只生成待审核知识，不会自动发布。

### 4.2 处理异常记录

重点查看：

- 品类待确认。
- 无聊天且无可用图片。
- 图片下载失败。
- MiMo调用失败。
- 聚类待人工确认。
- 低置信度候选。

未知品类不能手工改成手机兜底，应反馈给管理员补充品类映射。

### 4.3 审核主题候选

审核时依次检查：

1. 产品品类是否正确。
2. 是否将多个独立问题错误合并。
3. 是否将一个问题错误拆分。
4. 主标题是否自然、明确、可检索。
5. 知识正文是否只保留通用规则。
6. 是否错误外推单条工单结论。
7. 图例是否为脱敏且与知识点相关的案例图。
8. 推荐回复是否可以直接发送。
9. 当前新候选是否未主动生成标准关联；如已有标准关联，是否完整保留并进入搁置。
10. 证据不足时是否明确要求补充信息。

### 4.4 组员验证

组员主要填写：

- `是否值得沉淀`
- `是否可用`
- `如何修改`
- `问题反馈`

建议口径：

| 是否值得沉淀 | 含义 |
|---|---|
| 是 | 问题清楚、处理方式可复用，后续答疑有重复使用价值 |
| 否 | 仅适用于单个工单、没有通用处理信息或没有复用价值 |
| 未标注 | 尚未完成判断，不能进入批量送审 |

| 是否可用 | 含义 |
|---|---|
| 是 | 不修改即可使用 |
| 修改后可用 | 方向正确，但需要明确修改 |
| 否 | 主题、案例证据或知识内容存在根本错误 |
| 待补证据 | 当前证据不足，不能形成确定知识 |

`如何修改`应写明确修改动作，不要只写“优化一下”。

### 4.5 提交 CZ 终审

提交前确认：

- 数据已经脱敏。
- 品类正确。
- 已确认该知识点值得沉淀。
- 当前入口生成的新候选未主动填写关联标准项；已有标准关联候选应留在“标准关联搁置”，不要清空后送审。
- 标题和内容已确认。
- 图例和推荐回复已确认。
- 推荐回复已确认。
- 重复知识已经处理。

提交后查看：

- 已接收。
- 幂等复用。
- 已拒绝。
- Qwen3明确重复阻断。
- Qwen3疑似重复人工复核。

### 4.6 发布后验证

知识发布后，用真实业务问法执行检索：

- 是否能召回正确知识。
- 第一条结果是否合理。
- 推荐回复是否适用。
- 是否出现跨品类召回。
- 是否召回过期知识。

有问题时通过召回反馈写回，不直接修改模型输出日志。

## 5. 审核原则

### 可以通过

- 主题边界明确。
- 品类正确。
- 案例证据足以支持该知识。
- 内容可以被多个相同场景复用。
- 推荐回复不存在过度承诺。

### 必须修改或驳回

- 把相机镜头案例错误用于相机机身。
- 把未知品类默认归入手机。
- 将单条工单结论写成通用规则。
- 没有案例证据却输出确定结论。
- 标题只是关键词堆砌。
- 推荐回复包含内部审核术语。
- 多个独立问题被错误合并。

## 6. 自动审核说明

初次交付时自动审核应保持关闭。

只有满足以下条件的品类才可单独开启：

- 已积累足够人工验证样本。
- 准确率达到负责人确认的门槛。
- 自动放行精确率达到门槛。
- 已绑定具体模型版本。
- 已绑定具体 Prompt 版本。
- 已验证错误放行和回滚流程。

新品类不能继承其他品类的自动审核结果。

## 7. 常见问题

### 找不到品类

记录进入“待确认”。联系管理员增加品类名称、编码或别名，不要修改为其他相近品类。

### 找不到标准

确认：

- CZ 是否存在该品类标准。
- 标准是否已发布。
- 本次运行是否记录了标准快照版本。

### 模型没有运行

系统会使用规则兜底并标记人工审核。业务用户无需配置模型密钥，应联系管理员检查服务状态。

### 重复提交

使用相同幂等键重试时应显示“复用”，不会重复生成候选。若产生重复记录，应提交技术问题。

### 为什么不能直接自动发布

模型生成只是候选。最终发布前必须经过品类校验、标准校验、查重和审核，除非该品类已经完成正式自动审核验收。

- [ ] 统计标准Top 5和Top 1命中。

- [ ] 汇总新品类和标准缺口。
- [ ] 决定是否调整 Prompt、阈值或品类别名。
