# 2026-07-22 版本变更说明

## 2026-07-24 CZ 接通补充

- CZ 新版新增原生候选价值复核接口：
  `/api/v1/integration/knowledge-review-candidates:batch`。
- Answer Hub 自动化现在同步全部主题候选到该队列；待确认、风险和不值得沉淀项不会直接创建知识。
- CZ页面改用`/api/v1/integration/candidate-reviews`加载和编辑候选，
  人工完成后调用`/api/v1/integration/candidate-reviews:batch-submit`。
- 只有`ready`候选经过Qwen3查重后创建`review`待审核知识，系统不会自动发布。
- GitHub发布脚本仍只替换`prototypes/answer-hub`，不会覆盖CZ源码目录。

## 当前主链路

```text
第二部分批量脱敏记录
→ 无标准案例知识改写
→ 10项候选审核
→ 批量提交CZ
→ Qwen3语义查重拦截
→ CZ待审核与人工发布
```

## 主要变更

### 第三部分

- 默认不读取、不检索、不引用质检标准。
- 完整会话、历史实际回复和脱敏案例图作为主证据。
- 模型初标新增“是否值得沉淀”，区分值得沉淀、不值得沉淀和待确认。
- 组员验证新增“是否值得沉淀”；未确认值得沉淀或明确不值得沉淀的候选不会直接进入知识库批量送审，
  但会保留在CZ候选价值复核队列等待人工处理。
- 候选字段固定为：
  - 知识ID
  - 主标题
  - 副标题
  - 知识内容
  - 图例
  - 推荐回复
  - 知识分类
  - 关联标准项
  - 适用范围
  - 关键词
- `关联标准项`字段始终保留；当前流程不主动生成，新候选默认为空。
- 已有关联标准项、来源版本或正文标准引用保持原值，并进入“标准关联搁置”队列。
- 模型草稿出现标准引用时自动回退案例规则草稿。
- 模型不能因为缺少标准而否决候选。

### 第二部分批量接入

- `/api/v1/integration/second-part/records:batch`支持每批100条。
- 相同幂等键重试返回`reused`，不会重复调用模型。
- CZ服务端直接调用当前`answer_hub`无标准工作流生成主题候选。

### CZ批量导入

- 新增`/api/v1/integration/knowledge-review-candidates:batch`候选价值复核接入。
- CZ页面新增原生候选价值复核列表、人工编辑和“批量送审至知识库管理”按钮。
- 旧`/api/v1/topic-candidates/*`接口保留作兼容，不是新链路默认入口。
- 批量知识接入按候选逐条提交事务，单条向量生成、写库或提交失败不会导致整批回滚。
- 相同幂等键在批内和重试时均只创建一条知识。

### 服务端质量门禁

- 无标准候选统一规范为10项字段，但不删除或覆盖已有的`关联标准项`和`来源版本`。
- 通过或提交前必须具备知识ID、主标题、知识内容、推荐回复、知识分类、适用范围和关键词。
- 知识内容或推荐回复包含显式质检、回收或标准编号引用时保留原文，并搁置到后续标准关联流程；当前无标准入口不执行送审。
- 图例同时支持HTTP/HTTPS链接、外部媒体ID和空值；明确依赖图片但链接不可用时阻止提交。
- 图片写盘或最终事务失败时清理已写文件，避免孤立媒体。

### CZ插件化元数据

- CZ接入协议将知识改写执行器从`skill_name/skill_version`升级为`plugin_name/plugin_version`。
- 当前插件标识为`answer-hub-topic-transcription`，版本为`2026-07-22`。
- 旧Skill字段仅用于兼容已有调用，服务端会规范化为插件字段，不再写入新的处理元数据。

### Qwen3拦截

- `create`：正常进入CZ待审核。
- `review_duplicate`：进入待审核，但标记为疑似重复拦截。
- `block_duplicate`：明确重复，阻断入库。
- 批量响应增加`intercepted`和`blocked`计数。
- 自动化批量查重增加文本证据门禁：Qwen3标题和正文相似度达到阈值后，还必须有有效字符片段重合才能触发语义拦截。
- 手机、平板等同品类候选使用相同“先检查、再复测、记录结果”模板时，不会仅因模板相似被整批拦截。
- 完全重复、内容哈希一致和正文包含关系继续保留原拦截逻辑，不允许绕过Qwen3。

### 本地部署

- 基础`docker-compose.yml`已经包含PostgreSQL、Redis、CZ和CPU版Qwen3 Embedding。
- GPU环境通过`docker-compose.embedding-gpu.yml`覆盖。
- 新增：
  - `scripts/start_local_cz.ps1`
  - `启动本地CZ.cmd`
- 当前Docker镜像不再打包标准快照。
- 新增`scripts/verify_release.ps1`，统一执行测试、编译、前端语法、Compose和交付包安全检查。

## 安全边界

- 不提交`.env`、API Key、数据库文件、真实会话、运行日志和模型缓存。
- 所有第二部分记录必须先脱敏。
- 批量导入不得绕过Qwen3查重。
- 成功导入只进入`review`状态，不自动发布。
